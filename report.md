# Отчет по лабораторной работе №5: CQRS и Event Sourcing

**Цель работы:** Разработка микросервиса `balance-service` с использованием паттернов CQRS и Event Sourcing на базе Axon Framework и Axon Server.

## 1. Архитектура сервиса и инфраструктура (Axon Server)

* **Event Store и Event Bus:** В качестве решения для хранения событий и передачи команд/запросов использован **Axon Server**. Он обеспечивает надежное хранение последовательности событий и их маршрутизацию между Command и Query сторонами.
* **Администрирование:** Доступ к управлению событиями осуществляется через админ-панель Axon Server (`localhost:8024`).

> **Скриншот панели управления Axon Server (Standalone node):**
> <img src="screens/Standalone node.png" alt="Standalone node"/>

## 2. Реализация записи (Command Side)

* **Механизм:** Каждое изменение состояния (создание счета, пополнение, снятие) инициируется командой. Команда обрабатывается агрегатом, который генерирует соответствующее событие.
* **События:** События сохраняются в Axon Server в неизменяемом виде. Это позволяет просматривать историю транзакций и анализировать состояние системы на любой момент времени.

> **Просмотр событий в Axon Server (Search/Query):**
> <img src="screens/Search/Query.png" alt="Search/Query"/>


## 3. Модель чтения и проекции (Query Side)

* **Проекции:** Read-модель (`Balance`) обновляется через обработчики событий (`@EventHandler`). Она хранит только текущий результат, оптимизированный для быстрых ответов на GET-запросы.
* **Консистентность:** Данные в модели чтения обновляются сразу после того, как событие было успешно сохранено в Event Store.

## 4. Демонстрация работы API

Через Swagger UI были протестированы основные бизнес-сценарии:

1. **Создание баланса:** Посылка команды на инициализацию счета.
2. **Транзакции:** Выполнение нескольких операций `Deposit` и `Withdraw`.
3. **Проверка истории:** Извлечение списка всех событий (транзакций) по ID агрегата.

## Итог

В ходе работы была реализована система, в которой Axon Server выступает единым источником истины (Single Source of Truth). Использование CQRS позволило полностью разделить логику обработки команд и формирования ответов на запросы, а Event Sourcing обеспечил гарантированную сохранность истории всех банковских операций.
